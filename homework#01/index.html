Основні команди гіта

git config --global user.name blabla - при завантаженні git на пк, треба встановити ім'я користувача, дана команда дозволяє це зробити.
git config --global user.email blabal@bla - після встановлення імені, треба зробити те саме з електронною поштою, використавши цю команду.
git init - створює новий репозиторій.
git status - показує в якому стані файли. Тобто, можна побачити чи є зміни у файлах, які треба зробити контрольованими. Також, git status показує в якій гілці ви працюєте.
git add - після внесення змін до файлу, необхідно виконати цю команду, щоб почати його контролювати.
git commit - внести змінений/новий файл до репозиторію.
git log - показує історію комітів.
git branch - інструмент керування гілками. За допомогою цієї команди можна створити нову гілку, виводити список існуючих гілок, видаляти та змінювати їх ім'я.
git checkout - використовується для переключення на іншу гілку.
git merge - поєднує гілки в одну. Тобто, ця команда бере дві нові гілки та спільну для них батьківську та виконує їх злиття.
git push - надсилає вміст локальної гілки на віддалений репозиторій.
git pull - завантажує оновлення основної гілки на віддаленому репозиторії, до локальної гілки.
git clone - створює копію репозиторію.
.gitignore файл - це файл, в який додаються назви файлів, які не будуть виключенням, тобто вони не завантажуватимуться на віддалений репозиторій.

Advanced

git revert - створює новий коміт, в якому скасовує зміни певних попередніх комітів
git reset - пересуває основну гілку до попереднього коміта (використовуючі опцію -- mixed можна відмінити індексацію змін, а опція --hard зовсім їх видалити).
git rebase - поєднує гілки в одну. Але на відміну від git merge, git rebase бере коміти однієї гілки та застосовує їх до іншої гілки.
git cherry-pick - переміщує певні коміти з однієї гілки в іншу.
git stash - дозволяє зберегти незакомічені зміни в поточній робочій гілці. Це дозволяє переключатись між гілками проекту без виконання коміту для нових змін у файлах певної гілки.
